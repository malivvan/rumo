# ISSUES.md — Combined Architectural Issue Registry

Unified view of all architectural/systemic issues identified across the `rumo` project.
Each issue has a unique sequential ID. Sorted descending by severity (critical → high → medium → low).

**Sources:**
- `FIXME.md` — root package + `cmd` CLI
- `vm/FIXME.md` — VM, compiler, object system, concurrency
- `std/FIXME.md` — stdlib module framework + cross-module concerns
- `std/cli/FIXME.md` — CLI module (cobra-based)
- `std/cui/FIXME.md` — TUI module (tview-based)
- `std/shell/FIXME.md` — readline/shell module

**Severity:** critical (crashes, data races, build failures under normal use) · high (subtle bugs, silent misbehaviour) · medium (performance, maintainability, usability) · low (minor improvements)

**Related issues** are noted with `→ #N` cross-references.
